# Ampex D-175 EPROM Disassembly

Z80 disassembly listings and supporting reference material for the five EPROMs
dumped in `../EPROM_dumps/`. Of those, four are Z80 code (U4, U5, U6, U8); U52
is a character-generator bitmap and is out of scope for this tree.

## Regenerating

```sh
# Tools: z80dasm 1.1.6 (Debian/Ubuntu apt), Python 3, GNU make.
# On Windows hosts the Makefile auto-dispatches z80dasm through WSL.
make clean && make
```

Every `.asm` file is derived strictly from its `.bin` + (optional) `.sym`. Hand
edits belong in the `.sym` files, not the generated listings.

## File inventory

| File | Purpose |
|---|---|
| `Makefile` | Regenerates all listings + string extractions |
| `checksums.txt` | SHA-256 of every `.bin` in `../EPROM_dumps/` |
| `U4.asm` … `U8.asm` | z80dasm listings (regenerable) |
| `U4.sym` … `U8.sym` | Curated labels consumed by z80dasm (`-S`) |
| `strings_U*.txt` | Printable-ASCII runs with load-address and offset |
| `load_address_verification.md` | Phase 1 evidence that the origins in the Makefile are correct |
| `emulation_codes.md` | Phase 3 deliverable: rebuilt emulation-code reference |
| `xlate_tables.txt` | Full byte-by-byte reconstruction of all 11 per-emulation translation tables (regenerated by `../tools/dump_xlate_tables.py`) |
| `patch_records.txt` | Dump of every per-emulation patch record with addresses and cross-references (regenerated by `../tools/dump_patch_records.py`) |
| `handler_jump_table.txt` | Master 132-entry jump table at U5:0x20AA, resolved (regenerated by `../tools/dump_handler_table.py`) |
| `resolved_handlers.txt` | Per-emulation (command → handler-address) mapping using the master jump table (regenerated by `../tools/resolve_handlers.py`) |
| `handlers.md` | Semantic catalogue of classified handler entries: ~110 of 132 master-jump-table slots named; structural groups (cursor motion, erase, attribute, display-mode toggles, tab toggles, mode-bit pairs) documented. |
| `memory_map.md` | Phase 4 deliverable: ROM/RAM/IO map |

## Load addresses (verified Phase 1)

| ROM | File | Origin | Evidence |
|---|---|---|---|
| U4 | `U4-TMS2764JL.bin` (8 KB) | `0x0000` | Cold-start `DI ; IM 1 ; LD IX,3030h ; JP 243Dh` at 0x0000 jumps into U5 range. |
| U5 | `U5-TMS2764JL.bin` (8 KB) | `0x2000` | Quick-SET-UP banner `C=1=HDX 2=SILENT ...` at 0x23DF; U4 cold-start target 0x243D lands here. |
| U6 | `U6-AM2732-1DC.bin` (4 KB) | `0x4000` | Full SET-UP menu text at 0x4A9F; version string `AMPEX D175 TERMINAL V 3.5` at 0x4F00. |
| U8 | `U8-AM2732-1DC.bin` (4 KB) | `0xB000` | Star Technologies `Version No. 63.3 Copyright (C) 1983 Star Technologies, Inc.` at 0xB6E5; `JR +2Eh` at entry lands on `0xB030`. |

See `load_address_verification.md` for details.

## Relationship to `../agents.md`

`agents.md` is a prior hand-analysis. Hardware/chip identification from the PCB
photos is trustworthy; the emulation-code dispatch documentation in that file
contains significant errors and gaps (flagged by the project owner) and is being
rebuilt from scratch here as `emulation_codes.md`. Do not port dispatch claims
from `agents.md` without verifying each against this disassembly.
